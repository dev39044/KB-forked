# This document describes disaster recovery on Kerio Connect Mail server

1. You need to have a backup server | CHECK FIREWALL SETTINGS ON THIS SERVER!
2. You need to have rsynced C:\Program Files\Kerio & D:\Mail to your bkp server WITH DIFFERENT NAMES
E.g.: C:\Program Files\Kerio-synced & D:\Mail1
3. Install Kerio Connect on the bkp server on the same location at C:\Program Files\Kerio
4. YOU MUST CONFIGURE it right after installation!!! DO NOT CHANGE ANY FILE OR FOLDER BEFORE THIS STEP!

5. After you have configured your Mail server, then stop the service and do this:
	- rename C:\Program Files\Kerio\MailServer to MailServer-orig
	- then copy C:\Program Files\Kerio-synced\MailServer and paste in into original Kerio installation (THIS IS IMPORTANT TO PRESERVE WIN PERMS CORRECT)
	- rename D:\Mail to D:\Mail-orig (this step is relevant ONLY if you havn't forgotten to rename initially D:\Mail to D:\Mail1 folder!!!)
	- rename D:\Mail1 to D:\Mail
	
6. Now start Kerio Connect service and try to login and check users mailboxes.
7. Check your firewall and test the whole configuration if it works correctly.

8. DO NOT FORGET to stop & disable Kerio Connect service on your DR test server after successful test!
